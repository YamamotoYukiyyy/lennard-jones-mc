# Python と Rust の計算結果比較

- 日付: 2026-03-07
- 対象: アルゴン GEMC（Gibbs Ensemble Monte Carlo）

## 1. 同一条件での比較（N=100, cycles=5000）

| 項目 | Python | Rust |
|------|--------|------|
| **条件** | T*=0.873 (100K), cutoff=1.5, cell=1.5 | 同一 |
| **計算時間** | 716.2 秒 | 7.5 秒 |
| **sec/cycle** | 0.143 | 0.0015 |
| **速度比** | 1× | **約 95× 高速** |

### 最終状態

| 項目 | Python | Rust |
|------|--------|------|
| Box 1: N | 69 | 6 |
| Box 1: ρ* | 0.253 | 0.064 |
| Box 2: N | 31 | 94 |
| Box 2: ρ* | 0.510 | 0.392 |
| **rho_liq*** | 0.510 | 0.392 |
| **rho_vap*** | 0.253 | 0.064 |

### 受理率

| move | Python | Rust |
|------|--------|------|
| Displacement | 0.456 | 0.487 |
| Volume | 0.452 | 0.550 |
| Transfer | 0.068 | 0.081 |

### 解釈

- **数値の違い**: Monte Carlo は確率的な手法のため、Python と Rust で異なる乱数列（numpy vs rand）を用いるため、同一シードでも軌道が異なり、最終状態は一致しない。気液共存の範囲内で異なる実現が得られるのは正常。
- **受理率**: 両者とも同程度で、アルゴリズムの実装は妥当。
- **計算速度**: Rust は約 95 倍高速。
