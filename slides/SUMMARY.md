# スライド改善・テンプレート作成のまとめ

2D Lennard-Jones MC シミュレーションの発表スライド（Marp）の改善内容と、それに基づくテンプレートの使い方をまとめたドキュメントです。

---

## 1. 概要

- **対象**: `vibe_coding_slides.md`（README・CHANGELOG に基づく Vibe Coding 結果のスライド）
- **実施したこと**:
  - 1枚に情報を詰め込みすぎないよう分割・簡略化
  - 表をカード風にし、詳細は Appendix へ
  - 変更ログを「問題→対応→結果」などの型で統一
  - 物理モデルを実行方法より前に配置（理解の土台を先に）
  - 参照先を単独スライドから最終スライド下部へ統合
  - 上記の構成を反映したテンプレート `_template.md` を作成

---

## 3. デザイン方針・改善のポイント

### 3.1 情報の重さ

- **1枚1メッセージ**に絞る。視線が止まる余白を残す。
- 変更ログは (1/2)(2/2) に分けず、**1枚にまとめて短文**で収める。

### 3.2 表とカード

- 本編の表は**重要項目だけ**にし、**2×2 のカード風**の表で見せる。
- 全パラメータは **Appendix** に逃がす。

### 3.3 変更ログの型

- **やったこと → 内容 → 成果**（初回実装など）
- **問題 → 対応 → 結果**（不具合対応など）
- 複数項目を1枚にまとめる場合は **ラベル** 短文。の並び。

### 3.4 強調の役割

- **太字**: 結論・ラベル
- 等幅（コード）: 関数名・変数名・ファイル名に限定。多用しない。

### 3.5 実行方法

- 「仮想環境の Python で実行」と明示し、**簡潔かつ誤解のない**書き方にする。

### 3.6 参照先

- 単独スライドにせず、**最終スライド（仕様まとめ）の下部**に `<small>` で小さく置く。

### 3.7 スライドの順序

- **表紙 → 物理モデル（何を扱うか）→ 実行方法・パラメータ → 変更ログ → 現在の仕様 → Appendix**

## 4. テンプレート（_template.md）

### 4.1 使い方

1. `_template.md` をコピーし、新しいファイル名（例: `my_slides.md`）で保存する。
2. フロントマターの `title` / `description` を編集する。
3. `---`（ハイフン3つ）でスライドを区切り、内容を差し替える。
4. 不要なスライドは削除し、必要なスライドを追加する。
5. コマンドパレットで「Marp: Open Preview」でプレビュー。
6. プレビュー右上の「Export Slide Deck」から PDF / HTML / PPTX を出力。

### 4.2 テンプレートに含まれるパターン

| スライド | 用途 |
|----------|------|
| 表紙 | タイトル＋副題＋アジェンダ（箇条書き） |
| 本題 | モデル・背景（数式＋箇条書き） |
| 実行方法 | 仮想環境の Python で実行＋スクリプト名 |
| 重要パラメータ（4点） | 2×2 のカード風の表 |
| 変更ログ 1 | やったこと→内容→成果 |
| 変更ログ 2 | 問題→対応→結果 |
| 変更ログ 3・4 | **ラベル** 短文。の並び |
| 変更ログ 5 | 追加した機能／実装方法／得られる効果 |
| 現在の仕様はこの3点 | 番号リスト＋`<small>` で参照先 |
| Appendix | パラメータ一覧の表 |

### 4.3 テンプレート内のコメント

ファイル先頭の HTML コメントに、上記の使い方と**デザインのルール**（1枚1メッセージ、表のカード化、変更ログの型、参照先の置き方、推奨順序）を記載している。
