# Python → Rust 移植ノート

## 概要

2026-03-07 に Python版（`lennard_jones_mc/`）を Rust版（`lennard_jones_mc_rs/`）へ移植しました。
このドキュメントは、移植過程での主な変更点、修正内容、および未実装機能を記録しています。

## 移植の動機

1. **計算速度の大幅向上**: Python版は約50-100倍遅い
2. **複数温度VLE計算**: 複数条件での平衡計算が実用的な時間で可能に
3. **信頼性**: メモリ安全性を確保

## 実装した機能

✅ **3D GEMC (Gibbs Ensemble Monte Carlo)**
- 液相・気相の2つのボックスを並行管理
- 粒子移動・体積交換・粒子交換の各move

✅ **最適化技法**
- Cell List: O(N) または O(N^1.5) 計算量に削減
- Cutoff: r_c = 3σでのカットオフ

✅ **複数温度対応**
- 84K～148Kの自動VLE計算
- 各温度で200,000サイクル実行

✅ **統計出力**
- 生産相平均: equilibration 30%を除外、70%で計算
- 密度履歴CSV出力: サイクルごとの追跡
- 要約JSON出力: 最終統計値

✅ **単位変換**
- LJ無次元単位 → g/cm³への自動変換
- アルゴン固有のσ, εパラメータ使用

## 主な修正内容

### 2026-03-07: Cell List 重複排除バグ修正

**問題**: 修正前の Rust版では cell list の重複排除が不完全だった
- Volume move 受理率が極端に低い（1-2%）
- エネルギー計算に誤りがあった可能性

**修正内容**:
```rust
// 修正前（buggy）: ペア (i,j) と (j,i) が両方カウントされる
// 修正後（fixed）: i < j 条件でペアを一意化
```

**確認**: 修正後の volume move 受理率が 5-10% まで改善

**参考**:
- 修正前の結果ファイルは `results/archived/` に保存

## 未実装機能

❌ **長距離補正（Long Range Correction, LRC）**
- Lennard-Jones ポテンシャルの cutoff による誤差補正
- 文献値（Chialvo & Horita 2003）では適用
- 精度向上には必要だが、現在は実装されていない

❌ **受理率の適応的制御**
- Δ_max（粒子移動幅）や Δ_V_max（体積変化幅）を自動調整
- 目標受理率（例：50%）に自動収束
- 現在は固定値を使用

❌ **並列化**
- GPU計算やマルチスレッド化は未実装
- 大規模系（N > 1000）への対応

## 計算結果の信頼性

### Python vs Rust の比較

| 項目 | Python版 | Rust版 | 差異 |
|------|---------|--------|------|
| **液相密度** | ~0.70-0.75 g/cm³ | ~0.70-0.75 g/cm³ | ほぼ同一 |
| **気相密度** | ~0.001-0.003 g/cm³ | ~0.001-0.003 g/cm³ | ほぼ同一 |
| **Volume受理率** | ~5-10% | ~5-10% (修正後) | 修正で改善 |
| **実行速度** | 数分～数十分 | 数秒～数十秒 | 約50-100倍高速化 |

### 文献との比較

Chialvo & Horita (2003) の蒸気圧曲線と比較：
- LRC未実装のため、若干の系統的ずれ
- 気相密度が文献値より高い傾向
- 液相密度はほぼ一致

## パス依存性

移植時に注意が必要な点：
- Rust バイナリの出力先: `results/` ディレクトリに統一
- Python スクリプトの import: 相対 import に対応
- 解析スクリプト: `results/` 配下を参照するよう更新

## テスト・検証方法

1. **単一温度実行**: `cargo run --bin argon_compare`
2. **複数温度VLE**: `cargo run --bin argon_vle --release`
3. **ベンチマーク**: `python ../lennard_jones_mc_benchmark/src/benchmark_python_rust.py`
4. **結果の可視化**: `python analysis/plot_*.py`
